# Factory Skills

Skills are reusable capabilities that extend your AI assistant with specialized workflows and domain knowledge. They're the building blocks of your software factory.

## Featured Skills

### Documentation
- [Document Co-Authoring](/skills/01-doc-coauthoring) - Structured workflow for collaborative document creation with AI assistance

## What Makes a Good Skill?

A well-designed skill should be:

- **Focused**: Solves a specific problem well
- **Reusable**: Works across multiple projects and contexts
- **Composable**: Integrates easily with other tools
- **Well-documented**: Clear instructions and examples

## Skill Categories

**Documentation**: Creating, editing, and maintaining project documentation

**Testing**: Generating and running tests, analyzing coverage

**Code Review**: Analyzing code quality, suggesting improvements

**API Development**: Designing, documenting, and testing APIs

**Workflows**: Multi-step processes combining multiple skills

## Using Skills

Most skills can be invoked with a simple command in your AI development environment:

```
/skill-name
```

Some skills accept parameters or configuration options. Check individual skill documentation for details.
