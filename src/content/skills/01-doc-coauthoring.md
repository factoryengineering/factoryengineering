# Document Co-Authoring Skill

A comprehensive skill for creating high-quality documentation through an iterative, structured workflow.

## Features

- **Guided workflow**: Step-by-step process for document creation
- **Context gathering**: Efficiently capture requirements and background
- **Iterative refinement**: Multiple rounds of feedback and improvement
- **Reader verification**: Test documents with target audience

## Best For

- Technical specifications
- Decision documents
- Proposals and RFCs
- Long-form content

## How to Use

1. Invoke the skill: `/doc-coauthoring`
2. Provide context about your document
3. Review and refine through iterations
4. Verify with sample readers

## Example Workflow

```
User: /doc-coauthoring
AI: What type of document would you like to create?
User: A technical RFC for a new API design
AI: [Guides through structured workflow]
```

## Tips

- Prepare key points before starting
- Include relevant code samples or diagrams
- Iterate multiple times for best results
- Test with actual readers from your target audience
